# preview/

**Throwaway.** Одноразовый генератор для оценки контента и визуального разнообразия. Не часть основной архитектуры seobuilder3 — написан ad-hoc, чтобы быстро увидеть, как выглядит пул из 5 максимально разных сайтов.

## Что это делает

- Берёт 5 brief'ов на разных нишах (crypto / gardening / budgeting / dog-nutrition / piano-lessons).
- Каждому назначает свою **персону-писателя** (5 разных system prompt'ов для LLM).
- Каждому — свою **тему** (палетка / типографика / токены).
- Каждому — свою **композицию страницы** (разный набор и порядок секций).
- Генерирует контент через `gpt-4o-mini` (ключ из `~/.openclaw/.env`, шaренный с seobuilder2).
- Опционально — одно hero-изображение через `black-forest-labs/flux-schnell` (Replicate).
- Собирает по одной home-странице на сайт, инжектит CSS темы в `<head>`, пишет HTML в `out/site-{1..5}/index.html`.

## Чего НЕ делает

- Не собирает multi-page сайты. Только homepage — для быстрой оценки.
- Не делает мультиязычность. Только EN.
- Не трогает деплой, БД, identities, anti-fingerprint-валидатор. Это preview, не продукт.
- Не покрыт тестами. После оценки скорее всего будет удалён.

## Запуск

```bash
cd preview
pnpm i
pnpm generate          # с hero-изображениями (5 вызовов Replicate, ~10-30 сек)
pnpm generate:noimg    # без изображений (только текст, быстрее)
```

Результат: `preview/out/site-{1..5}/index.html` — открываются в браузере.

## Конфигурация

Ключи читаются из `~/.openclaw/.env` (тот же путь, что у seobuilder2):
- `OPENAI_API_KEY`
- `REPLICATE_API_TOKEN`

Если ключей нет — выпадает с осмысленной ошибкой.

## Почему template literals, а не preact+TSX

Preview — одноразовый. Template literals проще, заводятся без JSX-настройки, отлаживаются глазами. В основном коде seobuilder3 рендерер будет preact+TSX (одна из систем в пуле, §3.13 слой 5).
